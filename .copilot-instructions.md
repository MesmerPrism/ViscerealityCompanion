## ViscerealityCompanion — Copilot Instructions

### Build

```powershell
dotnet build ViscerealityCompanion.sln
dotnet test ViscerealityCompanion.sln
dotnet run --project src/ViscerealityCompanion.App
dotnet run --project src/ViscerealityCompanion.Cli -- --help
npm run pages:build
```

### Code Style

- Records with immutable fields for models. Use `with` for derived copies.
- Async-first: all I/O operations return `Task<T>` and accept `CancellationToken`.
- Factory pattern for external dependencies: each integration has a factory returning real or preview impl.
- Preview services: mock implementations that return `OperationOutcomeKind.Preview` without side effects.
- MVVM: `ObservableObject` + `AsyncRelayCommand` in `ViewModels/`. No code behind in `.xaml.cs` beyond initialization.

### Architecture

- `ViscerealityCompanion.Core` — platform-independent models, services, interfaces (targets `net10.0`).
- `ViscerealityCompanion.App` — WPF desktop shell (targets `net10.0-windows`).
- `ViscerealityCompanion.Cli` — CLI tool using `System.CommandLine` (targets `net10.0`).
- Public/private split: no private oscillator dynamics, twin backend coupling, or study presets in the public repo.

### Testing

- xUnit with `[Fact]` and `[Theory]`.
- Mock services via `Preview*` and `Fake*` implementations of interfaces.
- Test files go in `tests/ViscerealityCompanion.Core.Tests/`.
- Run with `dotnet test` or the test runner.

### External Dependencies

- **ADB**: Auto-detected via `QuestControlServiceFactory` from LocalAppData, ANDROID_SDK_ROOT, ANDROID_HOME, PATH.
- **liblsl**: P/Invoke to `lsl.dll`. Located via VISCEREALITY_LSL_DLL, app dir, or runtimes/win-x64/native.
- **Quest Session Kit catalogs**: Auto-discovered from VISCEREALITY_QUEST_SESSION_KIT_ROOT, AstralKarateDojo repo, or `samples/quest-session-kit/`.

### LSL Twin Protocol

- Command outlet: `quest_twin_commands` / `quest.twin` — sends `cmd <actionId>` frames.
- Config outlet: `quest_hotload_config` / `quest.config` — sends `begin → set key=value → end` snapshot frames.
- State inlet: `quest_twin_state` / `quest.twin` — receives `set key=value` readings from headset.
